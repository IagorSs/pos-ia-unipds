**Normalização de dados** é o processo de transformar os valores brutos de um conjunto de dados em uma escala consistente e compreensível para um modelo de machine learning. Em vez de trabalhar com os números originais — como idades de 25 a 40 anos —, aplicamos uma fórmula simples que os redimensiona para um intervalo específico, geralmente de 0 a 1.

### Como funciona (com um exemplo prático)

Suponha que temos as idades de três pessoas:

| Pessoa | Idade |
|--------|-----|
| Erick  | 30   |
| Ana    | 25   |
| Carlos | 40   |

Primeiro, identificamos a idade mínima (`idade_min`) e a máxima (`idade_max`):

- `idade_min = 25` (Ana)
- `idade_max = 40` (Carlos)

A fórmula de normalização que usamos é:

\[
\text{idade\_normalizada} = \frac{\text{idade} - \text{idade\_min}}{\text{idade\_max} - \text{idade\_min}}
\]

Aplicando-a a cada idade:

- **Ana**: \((25 - 25) / (40 - 25) = 0 / 15 = 0\)
- **Erick**: \((30 - 25) / (40 - 25) = 5 / 15 ≈ 0.33\)
- **Carlos**: \((40 - 25) / (40 - 25) = 15 / 15 = 1\)

Agora, nossos dados estão representados como valores normalizados:

```
[0.33, 1, 0, 0, 1, 0]   // Erick
[0,    0, 1, 0, 0, 1]   // Ana
[1,    0, 0, 1, 1, 0]   // Carlos
```

### Por que a normalização é necessária?

1. **Permite que o modelo aprenda padrões de forma mais eficaz** – Quando todas as características estão na mesma escala, o algoritmo não precisa ajustar seu peso internamente para lidar com grandes diferenças de magnitude. É como tentar correr em uma pista nivelada em vez de saltar de um chão para outro.

2. **Facilita a conversão para tensores** – A maioria dos frameworks de machine learning (incluindo TensorFlow.js) espera entradas como tensores numéricos. A normalização garante que esses tensores contenham números que se encaixam bem no intervalo de ativação do modelo.

3. **Melhora a estabilidade e a velocidade do treinamento** – Com valores normalizados, as atualizações dos pesos são mais estáveis, o que geralmente resulta em convergência mais rápida e em um modelo final mais confiável.

4. **Ajuda a evitar vieses** – Se uma característica varia de 0 a 1.000 e outra de 0 a 1, a primeira pode dominar o processo de aprendizado simplesmente por causa de seu tamanho. A normalização equaliza essa influência.

### Um paralelo do mundo real

Imagine que você está construindo um sistema de recomendação que considera dois atributos: **idade** (25–40) e **pontuação de fidelidade** (0–100). Se alimentarmos esses valores diretamente no modelo, a pontuação de fidelidade dominará as decisões de recomendação, pois seus números são muito maiores que os da idade. Ao normalizar ambos para o intervalo de 0 a 1, o modelo pode tratar os dois atributos de forma justa, assim como um juiz que dá notas iguais a diferentes competidores após ajustar suas pontuações.

### Resumo

- **Normalização** = redimensionar dados para um intervalo consistente (frequentemente 0–1).
- **Fórmula** = `(valor - min) / (max - min)`.
- **Objetivo** = tornar mais fácil para os algoritmos de machine learning aprender padrões, treinar modelos estáveis e evitar que características individuais dominem o processo de aprendizado.

Ao converter idades brutas (25, 30, 40) em idades normalizadas (0, 0,33, 1), preparamos os dados para que o modelo possa aprender de forma eficiente, prever resultados e, em última análise, fazer recomendações precisas. Esta etapa é um pré-processamento essencial que você encontrará em quase todos os projetos de machine learning.
