**O que são tensores?**

No TensorFlow.js, um *tensor* é simplesmente uma forma de armazenar números em uma estrutura que se assemelha a um vetor ou a uma lista em JavaScript.
Em vez de trabalhar com objetos complexos (como `{nome: "Erick", idade: 30, cor: "azul", localização: "São Paulo"}`), o TensorFlow.js precisa de uma representação que contenha **apenas números**. Esses números são organizados em um ou mais níveis de arrays — o que chamamos de tensor.

**Como eles são representados em JavaScript**

Imagine que você tem um array simples de números:

```js
const numeros = [0.5, 1.2, 3.0];
```

Esse array unidimensional é um tensor de *ordem 1* (um vetor). Se você precisar de uma estrutura mais rica, pode aninhar arrays, criando um tensor bidimensional (uma matriz):

```js
const matriz = [
  [1, 0, 0],   // vetor para Erick (azul)
  [0, 1, 0],   // vetor para Ana   (vermelho)
  [0, 0, 1]    // vetor para Carlos (verde)
];
```

Cada linha interna é um vetor numérico que representa uma única observação. No exemplo do contexto, as três cores (`azul`, `vermelho`, `verde`) são codificadas como **one‑hot vectors**:

* `azul` → `[1, 0, 0]`
* `vermelho` → `[0, 1, 0]`
* `verde` → `[0, 0, 1]`

Essas linhas, juntas, formam um tensor 3×3 que o TensorFlow.js pode alimentar a um algoritmo de aprendizado.

**Por que precisamos dessa conversão?**

O TensorFlow.js *não* entende objetos JavaScript ou seus tipos de dados primitivos diretamente. Para que o modelo possa aprender, você deve:

1. **Extrair os campos numéricos relevantes** (por exemplo, a cor preferida).
2. **Transformar os dados categóricos em números** (a codificação one‑hot faz isso).
3. **Organizar os números em um tensor** (um ou mais arrays aninhados).

**Exemplo prático**

Supondo que você queira treinar um modelo para prever a categoria (`premium`, `medium`, `basic`) com base na cor preferida, você poderia criar um tensor como este:

```js
const tensorCores = tf.tensor2d([
  [1, 0, 0],  // azul  → premium
  [0, 1, 0],  // vermelho → medium
  [0, 0, 1]   // verde  → basic
]);
```

Agora `tensorCores` é um tensor bidimensional que o TensorFlow.js pode usar para treinamento ou inferência.

**Resumo**

- **Tensor = array numérico** (um ou mais níveis de arrays no JavaScript).
- Eles são usados porque os algoritmos de ML só entendem números.
- No TensorFlow.js, você cria um tensor a partir de dados JavaScript simples, convertendo primeiro os dados categóricos (como cores) em números (por exemplo, codificação one‑hot) e, em seguida, organizando esses números em uma estrutura de array aninhado.

Essa é a ideia básica! Quando você vir um tensor no TensorFlow.js, pense em um vetor ou matriz simples de números prontos para serem processados por um modelo de aprendizado.
