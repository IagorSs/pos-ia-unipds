**One‑Hot Encoding – o que é e quando usá-lo**

One‑Hot Encoding é uma maneira de transformar dados categóricos (como “cor favorita” ou “localização”) em um formato numérico que pode ser alimentado a um modelo de machine learning. Em vez de dar ao algoritmo uma única palavra como rótulo, criamos um vetor binário onde **apenas uma posição é 1** e todas as outras são 0. Essa representação deixa claro qual categoria está presente, sem sugerir uma ordem ou magnitude entre as categorias.

### Como o exemplo do contexto ilustra isso

Imagine que temos três pessoas e queremos descrever cada uma delas usando números:

```js
const pessoas = [
  { nome: "Erick", idade: 30, cor: "azul",   localizacao: "São Paulo" },
  { nome: "Ana",   idade: 25, cor: "vermelho", localizacao: "Rio" },
  { nome: "Carlos", idade: 40, cor: "verde",  localizacao: "Curitiba" }
];
```

As colunas categóricas (`cor` e `localizacao`) são transformadas em vetores one‑hot:

| Pessoa | azul | vermelho | verde | São Paulo | Rio | Curitiba |
|---------|------|---------|-------|-----------|-----|----------|
| Erick   | 1    | 0       | 0     | 1         | 0   | 0        |
| Ana     | 0    | 1       | 0     | 0         | 1   | 0        |
| Carlos  | 0    | 0       | 1     | 0         | 0   | 1        |

No código do contexto, isso aparece como:

```js
const tensorPessoas = [
  [30, 1, 0, 0, 1, 0], // Erick
  [25, 0, 1, 0, 0, 1], // Ana
  [40, 0, 0, 1, 1, 0]  // Carlos
];
```

* Os dois primeiros números após a idade são o **one‑hot para “cor”** (azul, vermelho, verde).
* Os três últimos números são o **one‑hot para “localização”** (São Paulo, Rio, Curitiba).

### Quando usar One‑Hot Encoding

1. **Variáveis categóricas verdadeiras** – quando um campo só pode assumir um de vários valores distintos, sem ordem inerente (ex.: cores, cidades, categorias de produtos).
2. **Modelos que esperam entradas numéricas** – a maioria dos algoritmos de machine learning (incluindo redes neurais em TensorFlow.js) só trabalha com números, então o one‑hot encoding permite que esses dados sejam usados diretamente.
3. **Mutuamente exclusivo** – cada observação pertence a **apenas uma** das categorias (você não pode estar “meio azul e meio vermelho”). O vetor binário reflete perfeitamente essa exclusividade.

### Um exemplo rápido do contexto

No trecho do documento, eles mencionam:

> “para cor e localização fixamos zero e um, a fins de curiosidade, isso é chamado de **one hot take**, quando só pode ser uma das opções”

Isso mostra exatamente o caso de uso: tanto “cor” quanto “localização” são categorias mutuamente exclusivas, então cada uma é representada por um vetor one‑hot.

### Resumo

- **One‑Hot Encoding** = vetor binário onde apenas uma entrada é 1.
- Útil para transformar rótulos categóricos (ex.: “azul”, “vermelho”, “verde”) em números que um modelo possa entender.
- Use quando você tem dados categóricos que são **mutuamente exclusivos** e o algoritmo espera entrada numérica.

Ao converter dados como `cor` e `localizacao` em um tensor one‑hot, você prepara o conjunto de dados para treinamento com TensorFlow.js, permitindo que o modelo aprenda padrões sem ambiguidade.
