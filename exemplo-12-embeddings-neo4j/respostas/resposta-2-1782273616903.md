### Como transformar objetos JavaScript em tensores no TensorFlow.js

O TensorFlow.js só consegue trabalhar com **números puros** – ele não entende diretamente objetos JavaScript como `{ nome: "Carlos", idade: 40, cor: "verde", localizacao: "Curitiba" }`.
Para usar esses dados em um modelo, primeiro precisamos **extrair os valores numéricos** (ou codificar as categorias) e, em seguida, agrupá-los em um *tensor*.

Abaixo está um exemplo passo a passo usando os dados que você forneceu.

---

#### 1️⃣ Comece com o array de objetos

```js
const pessoas = [
  { nome: "Carlos", idade: 40, cor: "verde", localizacao: "Curitiba" },
  { nome: "Erick",  idade: 30, cor: "azul",  localizacao: "São Paulo" },
  { nome: "Ana",   idade: 25, cor: "vermelho", localizacao: "Rio" }
];
```

O TensorFlow.js não consegue processar este array diretamente, então o transformamos em uma estrutura numérica.

---

#### 2️⃣ Escolha quais propriedades se tornam características

| Propriedade | Tipo      | Como codificamos |
|----------|-----------|--------------|
| `idade`  | numérica  | mantemos o número bruto |
| `cor`    | categórica| **one‑hot** (azul, vermelho, verde) |
| `localizacao` | categórica | **one‑hot** (ex.: São Paulo, Rio, Curitiba) |

> **Por que one‑hot?**
> One‑hot transforma um valor categórico em um vetor de 0s e 1s, onde apenas a posição correspondente à categoria recebe um 1. É uma maneira perfeita de representar dados categóricos como números.

---

#### 3️⃣ Mapeie os valores categóricos para índices

```js
// Todas as cores possíveis, na ordem que usaremos para one‑hot
const cores = ["azul", "vermelho", "verde"];

// Todas as localizações possíveis
const localizacoes = ["São Paulo", "Rio", "Curitiba"];
```

Agora podemos converter cada cor e localização em um vetor numérico.

---

#### 4️⃣ Crie os tensores

```js
import * as tf from '@tensorflow/tfjs';

// 4a. Idade → uma coluna (3 amostras)
const idades = tf.tensor1d(pessoas.map(p => p.idade), 'int32');

// 4b. One‑hot para a cor
const coresIdx = pessoas.map(p => cores.indexOf(p.cor));   // [2, 0, 1]
const corOneHot = tf.oneHot(coresIdx, cores.length);       // shape [3, 3]

// 4c. One‑hot para a localização
const locIdx = pessoas.map(p => localizacoes.indexOf(p.localizacao)); // [2, 0, 1]
const locOneHot = tf.oneHot(locIdx, localizacoes.length);             // shape [3, 3]

// 4d. Combine tudo em uma única tabela 2D (amostra × característica)
const tabelaFeatures = tf.concat(
  [idades.reshape([3,1]), corOneHot, locOneHot],   // concatenar ao longo do axis=1
 1
);

/* Resultado (aproximado):
   [[40, 0, 0, 0, 1, 0],
    [30, 1, 0, 0, 0, 0],
    [25, 0, 1, 0, 0, 1]]
*/
```

**Explicação**

| Coluna | Significado |
|--------|---------|
| `0` | `idade` (numérica) |
| `1‑3` | codificação one‑hot para `cor` (azul, vermelho, verde) |
| `4‑6` | codificação one‑hot para `localizacao` (São Paulo, Rio, Curitiba) |

Agora `tabelaFeatures` é um **tensor 2D** (`[3,6]`) que o TensorFlow.js pode alimentar em um modelo.

---

#### 5️⃣ (Opcional) Remova as intermediateas para liberar memória

```js
idades.dispose();
coresIdx = null;          // permitir que o GC limpe
locIdx = null;
```

---

### Resumo rápido

| Passo | O que fazer | Por quê |
|------|------------|-----|
| **Extrair** valores numéricos das propriedades do objeto | `pessoas.map(p => p.idade)` | O TensorFlow.js só entende números |
| **Codificar** categorias (one‑hot) | `tf.oneHot([...], tamanhoDoVocabularo)` | Transforma texto em vetor numérico |
| **Agrupar** em um tensor 2D | `tf.concat([...], axis=1)` | Modelo espera uma tabela de características |
| **Limpar** intermediários | `.dispose()` | Evita vazamentos de memória |

---

### Analogia do mundo real

Pense em um objeto JavaScript como uma **foto** de uma pessoa: você vê nome, idade, cor favorita, etc., mas o computador só entende **pixels** (números). Converter a foto em um tensor é como **reconhecer os recursos** (olhos, nariz, boca) e armazenar cada recurso como um número. A codificação one‑hot é como criar um **mapa binário** onde apenas uma cor (ou localização) acende como “1” e as outras ficam como “0”.

---

### TL;DR

* O TensorFlow.js **não** entende objetos JavaScript.
* Converta cada propriedade em dados numéricos:
  * Mantenha números brutos (`idade`).
  * Transforme categorias em vetores one‑hot (`cor`, `localizacao`).
* Combine os vetores numéricos em um **tensor 2D** (`tf.tensor2d` ou `tf.concat`) que pode ser alimentado diretamente em um modelo.

É só isso! Com esses passos, seus dados JavaScript estão prontos para serem usados em qualquer modelo do TensorFlow.js. Boa programação! 🚀
